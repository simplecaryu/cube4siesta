# Issue #1: cross-pseudo ρ restart via difference density

## Status
Open — **implementation in progress** on branch `feat/diff-density-restart`.

## Problem

`Rho.Restart` as implemented today overwrites SIESTA's `DRho` with a total
valence density ρ(r) read from an external .RHOIN file. When the external
density was produced by a DFT code whose pseudopotential partitions the
core/valence boundary differently from SIESTA's .psf, the scalar field
stored in the file is simply **a different physical quantity** from what
SIESTA would itself compute for the same system.

Concrete case (`examples/vsse_openmx/`):

| | OpenMX V_PBE19 | Cornell NNIN `V-gga.psf` |
|---|---|---|
| V valence config | 3s² 3p⁶ 3d³ 4s² | 3d³ 4s² |
| V valence electrons | 13 | 5 |
| ρ(r) contains V 3s/3p peak? | **yes** | **no** |

Result of injecting OpenMX `tden.cube` into SIESTA with the 5-valence
V.psf:
- ∫ρ dV comes out at 25 e, SIESTA expects 17 e
- a crude `--rescale-to 17.0` squashes the entire field by 0.68×,
  dragging the semicore peak along with the bonding density
- one-shot diag → Fermi = +30 eV, E_KS off by ~150 eV; garbage.

This is **not** a file-format issue (the Cornell .psf loads fine,
baseline SCF converges to E_KS = -696 eV in ~32 steps).

## Why "just ignore the core part" is non-trivial

OpenMX's `tden.cube` is a single scalar field — no shell decomposition.
Without an analytic model of the 3s/3p contribution we cannot subtract
it from the cube.

## Proposed fix — transfer the *difference density*

Decompose
$$\rho_\text{scf}(r) = \rho_\text{atomic}(r) + \Delta\rho(r)$$

- `ρ_atomic(r)` = sum of spherically-averaged isolated-atom densities.
  **Strongly pseudo-dependent** (differs by exactly the semicore peak
  between 5-val and 13-val V).
- `Δρ(r)` = bonding-induced redistribution. Lives in the valence
  shells that all reasonable pseudos agree on (V 3d/4s, S 3s/3p,
  Se 4s/4p). **Approximately pseudo-independent** at chemically
  relevant resolution.

Cross-code injection then uses:
$$\rho_\text{target}(r) \approx \rho_\text{atomic}^\text{SIESTA}(r) + \Delta\rho^\text{source}(r)$$

- The atomic part is generated by SIESTA itself from its own `.psf`
  (`rhoatm` array, already present in `dhscf.F`).
- The difference part is what's transferred across codes. Each source
  code can provide it in a natural way:
  - **OpenMX**: `*.dden.cube` is already exactly this (integrates to
    ~0 as expected).
  - **VASP**: CHGCAR from the SCF run, minus an ICHARG=1/NELM=0
    CHGCAR (atomic-superposition start, no SCF).

## Design

### SIESTA patch
- Add `Rho.Restart.Diff` fdf logical (default `.false.`). Only takes
  effect when `Rho.Restart` is `.true.`.
- In `dhscf.F`, when both flags are on, after
  `inject_rho_from_file(DRho, ...)` has loaded Δρ, add SIESTA's own
  `rhoatm` to each diagonal spin component — same pattern as the
  existing `read_deformation_charge_cdf` branch (L903).

### cube4siesta CLI
- `cube4siesta convert --from-diff-cube dden.cube --output vsse.RHOIN.diff ...`
  stores Δρ (can be negative) in the .RHO format. No rescaling or
  sign flipping — the SIESTA side handles the addition of rhoatm.
- For VASP: new helper `vasp_diff_chgcar(scf, atomic, out)` that reads
  two CHGCARs and writes their volume-normalized difference as a cube.

### Expected qualitative win
- VSSe: Fermi + E_KS should come into a sane range (order of SIESTA's
  own baseline ±O(eV)), rather than +30 eV / +150 eV off.
- SiC: should improve on the current 0.5 eV residual, provided we can
  produce the atomic-superposition CHGCAR (re-run VASP with ICHARG=1
  NELM=0, same structure).

## Out of scope here
- Shell-resolved density transfer (would need projectors).
- Full frozen-core reconstruction a la PAW augmentation.
